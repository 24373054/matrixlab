# HTTP 502 错误解决方案

## 问题分析
您的Jekyll服务器正在运行，但无法通过公网IP `140.143.183.163:4000` 访问，原因如下：

1. **服务器状态**: Jekyll服务器正在端口4000上运行（绑定到0.0.0.0）
2. **网络配置**: 
   - 私有IP: 10.2.24.4
   - 公网IP: 140.143.183.163
3. **问题原因**: 云服务提供商的安全组/防火墙阻止了端口4000的入站流量

## 解决方案

### 方法1：配置安全组规则（推荐）
在您的云服务提供商控制台中，为实例的安全组添加以下规则：

**入站规则**:
- 协议: TCP
- 端口范围: 4000
- 源IP: 0.0.0.0/0 (或特定IP范围)
- 描述: Jekyll开发服务器

### 方法2：使用标准HTTP端口（80端口）
修改Jekyll配置使用标准HTTP端口：

```bash
# 停止当前服务器
pkill -f jekyll

# 使用端口80重新启动（需要root权限）
cd yz/Matrix_Lab/Matrix_Lab1.0
sudo jekyll serve --host 0.0.0.0 --port 80
```

### 方法3：使用反向代理
配置Nginx作为反向代理：

1. 安装Nginx:
```bash
sudo yum install nginx -y  # 对于CentOS/RHEL
# 或
sudo apt install nginx -y  # 对于Ubuntu/Debian
```

2. 创建Nginx配置文件 `/etc/nginx/conf.d/jekyll.conf`:
```nginx
server {
    listen 80;
    server_name 140.143.183.163;
    
    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

3. 重启Nginx:
```bash
sudo systemctl restart nginx
```

## 验证步骤
配置完成后，通过以下方式验证：

1. 本地访问: `http://localhost:4000`
2. 公网访问: `http://140.143.183.163:4000` (或端口80)
